<context>
Current unit-ownership.ts uses SQLite for atomic claim/unclaim operations.

File format for .claim files:
```json
{
  "unitId": "U001",
  "claimedBy": "agent-abc",
  "claimedAt": "2026-04-14T10:00:00Z",
  "status": "claimed"
}
```

Atomic write pattern:
1. Write to temp file (.claim.tmp)
2. fsync()
3. rename(.claim.tmp, .claim)

This is atomic on POSIX systems. On Windows, use MoveFileEx with MOVEFILE_REPLACE_EXISTING.
</context>
