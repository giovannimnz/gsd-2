<context>
Current memory/storage.ts uses sql.js (WASM SQLite in memory).

Tables:
- jobs: id, phase, status, created_at, updated_at
- threads: id, status, cwd, created_at, updated_at
- stage1_outputs: id, job_id, output, created_at

File structure:
```
.gsd/memory/
├── jobs/
│   ├── job-001.json
│   └── job-002.json
├── threads/
│   └── thread-001.json
└── stage1/
    └── output-001.json
```

Each JSON file contains the full row data. Indexes are replaced by directory scans (acceptable for typical job/thread counts).
</context>
